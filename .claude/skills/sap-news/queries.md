# Шаблоны поисковых запросов

Все запросы запускать параллельно одним вызовом (несколько WebSearch в одном response). Год в запросах — текущий; при поиске по прошлому периоду подставлять соответствующий.

## Режим по умолчанию / --from / --to (широкий поиск)

Базовый набор из 8 запросов:

1. `переход с SAP на российское ПО {год} компании`
2. `импортозамещение SAP Россия {год} новости`
3. `замена SAP ERP российская альтернатива {год}`
4. `миграция SAP 1С ERP внедрение {год} крупная компания`
5. `"уход SAP" Россия замена системы {год}`
6. `SAP Россия отказ переход Галактика Турбо ELMA {год}`
7. `рынок ERP Россия {год} замещение SAP Oracle`
8. `CNews TAdviser переход с SAP {год}`

Если ищем за узкий период (неделю-две) — добавить в 2–3 запроса слово `{месяц}` или формулировку «за последнюю неделю».

## Режим --company <имя>

Для конкретной компании:

1. `{компания} SAP замена переход {год}`
2. `{компания} импортозамещение ERP {год}`
3. `{компания} Global ERP 1С Галактика внедрение`
4. `site:cnews.ru OR site:tadviser.ru OR site:comnews.ru {компания} SAP`

Если компания многословная (Сургутнефтегаз, РусГидро) — в запросе без кавычек. Если в названии редкое слово (ФосАгро) — можно с кавычками.

## Правила работы с запросами

1. **Всегда актуальный год** — подставлять текущий год в шаблоны. Если ищем за январь 2026 — подставлять 2026. Если за декабрь 2025 — `{год}` = 2025.
2. **Никогда не запускать больше 8 запросов за раз в параллели** — это перегружает и даёт много дублей.
3. **Если первый прогон дал < 5 новых новостей** — добавить 2–3 уточняющих запроса по именам конкретных компаний из раздела «Компании под особым наблюдением» в `sources.md`.
4. **Операторы `site:`** — использовать умеренно, они сужают выдачу. Для широких прогонов — без `site:`, для уточнения по конкретным источникам — с `site:`.
5. **Кавычки в запросах** — только вокруг уникальных фраз («уход SAP», «замена SAP»). Не заключать в кавычки длинные фразы — выдача обнулится.
