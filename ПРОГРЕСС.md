## TL;DR

Система **работает end-to-end**. Еженедельный дайджест о переходе российских компаний с SAP на отечественные решения собирается автоматически по понедельникам в 10:00 МСК и рассылается на два email. Ручное вмешательство нужно только когда захочется изменить источники, шаблоны запросов или получателей.

## Что сделано (хронология)

1. **Разведка источников.** Прогнали 20 поисковых запросов по теме «переход с SAP → отечественные ERP». Отобрали 6 категорий источников (от must-have CNews/TAdviser/ComNews/Ведомости/РБК до шумовых «топов аналогов SAP», которые отсекаются).
2. **Скил `/sap-news`.** Написан в `.claude/skills/sap-news/` — три файла: `SKILL.md` (алгоритм), `sources.md` (каталог источников), `queries.md` (шаблоны запросов для режимов default / --company / --sector).
3. **Первый дайджест вручную.** Собран за 2026-01-01 .. 2026-04-23, 16 новостей — `news-archive/digest-2026-04-23.md`. Это «посев» архива для дедупликации.
4. **Архив URL.** `news-archive/seen-urls.txt` с 19 URL (16 из первого дайджеста + 3 из второго ручного).
5. **GitHub-репозиторий.** https://github.com/abdublinn/sap-news — публичный, ветка `main`.
6. **Cloud routine.** `trig_01KR7hpZ8jJ4BBNPQKSpz7xU` (Weekly SAP news digest). Cron `0 7 * * 1` UTC = понедельник 10:00 МСК. Модель claude-sonnet-4-6. Работает через `/web-setup` auth + флаг «Allow unrestricted git push».
7. **GitHub Action для email.** `.github/workflows/notify-digest.yml` — на push `news-archive/digest-*.md` отправляет письмо через Gmail SMTP (action `dawidd6/action-send-mail@v4`).
8. **Gmail-настройка.** App password в GitHub Secrets (`GMAIL_APP_PASSWORD`), отправитель `GMAIL_USER=ab.dublinn@gmail.com`, получатели в `EMAIL_RECIPIENTS=ab.dublinn@gmail.com,M.Skovorodin@exrp.ru`.
9. **Второй дайджест (тестовый).** Собран локально за 2026-04-17 .. 2026-04-24, 4 новости — `news-archive/digest-2026-04-24.md`. На этом дайджесте проверена работа Action + email.

## Уроки, которые мы усвоили (для контекста)

Короткая выжимка — чтобы не повторять путь через грабли в следующем проекте:

1. **Cloud routines по умолчанию пушат только в ветки `claude/*`.** Для main нужен per-repo тоггл «Allow unrestricted git push» в UI https://claude.ai/code/routines/{id} → Edit. Флаг не пишется через API (HTTP 400), только через UI.
2. **`/web-setup` в CLI** — даёт routines read-auth (clone). Без этого даже clone не пройдёт.
3. **Gmail-коннектор claude.ai не умеет send_email** — только `create_draft`, `list_drafts`, `search_threads`, метки. Для автоматической отправки использовать GitHub Action + SMTP (Gmail app password).
4. **OneDrive + git.** Папка проекта лежит в OneDrive. Пока работает без конфликтов, но на крупных операциях (многочасовые rebase) возможны редкие артефакты. Если начнёт глючить — перенести репо в `C:\Users\adubl\projects\`.
5. **Даты в TAdviser** почти всегда неочевидны — раздел «Проект:...» обновляется регулярно, а первая публикация не указана явно. Если нужна точная дата — искать в cnews.ru/vedomosti.ru, которые обычно её указывают в URL.
6. **WebFetch из cloud-агента** часто падает с 403 на российских сайтах (IP-блокировка). WebSearch при этом работает — через какой-то внутренний API. Скил адаптируется, но качество дайджеста зависит от того, сколько источников сумел прочитать.

## Если что-то сломалось / не пришло письмо

Диагностика по порядку:

1. **Письмо не пришло в понедельник** → проверить последний run routine: https://claude.ai/code/routines/trig_01KR7hpZ8jJ4BBNPQKSpz7xU
   - Статус `Succeeded`? → смотреть коммиты в main: должен быть `Weekly SAP news digest 2026-04-27`
   - Коммит есть? → смотреть Action: https://github.com/abdublinn/sap-news/actions
   - Action зелёный? → проверять спам, потом Gmail Postmaster
   - Action красный? → открыть логи, скорее всего истёк `GMAIL_APP_PASSWORD` — пересоздать и обновить секрет
2. **Routine упал** → скриншот лога из UI routine в новую сессию + попросить Claude разобрать
3. **Всё ок, но дайджест пустой/однообразный** → возможно, нужно подкрутить запросы (`.claude/skills/sap-news/queries.md`) или добавить источники (`.claude/skills/sap-news/sources.md`). Правки в репо → routine подхватит при следующем запуске.
